
# iptable封闭端口

需要书写iptables文件，对端口阻止外部所有ip访问，并对某些ip列表进行开放，并且需要将开放的ip列表放到一行里面
好的，以下是一个iptables规则的示例，用于阻止所有外部IP地址访问特定端口，并对某些IP地址列表进行开放，且开放的IP地址列表放在一行里面：
# 阻止所有外部IP地址访问特定端口
iptables -A INPUT -p tcp --dport <端口号> -j DROP

# 允许特定IP地址访问端口
iptables -A INPUT -p tcp -s <IP地址1>,<IP地址2>,<IP地址3> --dport <端口号> -j ACCEPT

# 保存规则
service iptables save
请将<端口号>替换为您要限制的实际端口号，并将<IP地址1>, <IP地址2>, <IP地址3>替换为您要开放的特定IP地址。确保您具有适当的权限来运行这些命令，通常需要以root用户身份执行。
这些规则将阻止所有外部IP地址访问特定端口，但允许您在<IP地址1>, <IP地址2>, <IP地址3>中指定的IP地址进行访问。您可以根据实际需求添加或修改开放的IP地址列表。

在编写iptables规则后，可以使用iptables-save > /root/iptables命令将规则保存到文件中。这个命令将把当前的iptables规则导出到指定的文件中，以便在需要时进行恢复或重新加载。
请注意，执行此命令需要具有适当的权限，通常需要以root用户身份执行。确保指定的文件路径是正确的，并且您具有在该路径下写入文件的权限。
如果您想在重新启动机器后自动加载iptables规则，可以将此命令添加到开机启动脚本中，例如/etc/rc.d/rc.local。
另外，如果您需要恢复保存的规则，可以使用iptables-restore < /root/iptables命令从文件中加载规则。确保在加载规则之前，您的iptables配置文件是空的或不存在，否则可能会覆盖现有的规则。
iptables-restore < /root/iptables
